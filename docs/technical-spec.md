# Техническое задание

## Название

Минимальный MCP reference-проект для подключения клиента к локальному серверу и получения списка инструментов.

## Цель

Собрать маленький расширяемый sandbox-проект, который демонстрирует базовый lifecycle работы с MCP:

1. запуск локального MCP server;
2. подключение MCP client;
3. выполнение `initialize`;
4. выполнение `tools/list`;
5. вывод списка доступных инструментов в консоль.

## Контекст

Проект создаётся как учебный reference, но с прицелом на дальнейшее развитие. Поэтому решение должно быть достаточно простым для быстрого старта и при этом не мешать в будущем:

- выносить сервер наружу;
- заменять локальный endpoint на удалённый;
- добавлять новые инструменты и сценарии проверки.

## Функциональные требования

- В проекте должен быть собственный MCP server.
- Сервер должен подниматься локально и предоставлять MCP endpoint по HTTP-based transport.
- Сервер должен объявлять минимум 1-2 простых инструмента, например `ping` и `echo`.
- В проекте должен быть собственный MCP client.
- Клиент должен подключаться к локальному MCP endpoint сервера.
- Клиент должен проходить базовый handshake MCP и выполнять `initialize`.
- После инициализации клиент должен вызывать `tools/list`.
- Клиент должен печатать в консоль список доступных инструментов в читаемом виде.

## Нефункциональные требования

- Код должен быть организован так, чтобы transport-логика не была жёстко смешана с остальной логикой примера.
- Endpoint сервера должен быть вынесен в конфигурацию, константу верхнего уровня или другой явно редактируемый параметр.
- Решение должно оставаться минимальным: без лишних слоёв абстракции, которые не нужны для базового сценария.
- Для нового публичного кода и важного orchestration-кода должна быть добавлена документация.

## Ожидаемая структура решения

- Модуль или пакет сервера с минимальной регистрацией MCP tools.
- Модуль или пакет клиента, который выполняет подключение и `tools/list`.
- Точка входа или команды запуска для сервера и клиента.
- README с краткой инструкцией по запуску.

## Сценарий проверки

1. Запустить локальный MCP server.
2. Запустить MCP client.
3. Убедиться, что соединение установлено без ошибок.
4. Убедиться, что `initialize` проходит успешно.
5. Убедиться, что `tools/list` возвращает зарегистрированные инструменты.
6. Убедиться, что клиент выводит их список в консоль.

## Критерии готовности

- есть отдельный sandbox-проект;
- в проекте реализован локальный MCP server;
- сервер работает по HTTP-based transport;
- клиент успешно подключается к серверу;
- клиент получает и выводит список инструментов;
- для переноса сервера на удалённый хост достаточно в основном поменять URL и инфраструктурное окружение, а не переписывать принцип подключения клиента.

## Ограничения и ориентиры

- На текущем этапе не требуется полноценная бизнес-логика.
- На текущем этапе не требуется сложная авторизация, хранение данных или production-hardening.
- Если стек ещё не выбран окончательно, предпочтение отдаётся варианту, который быстрее даст рабочий end-to-end результат без потери расширяемости.

## Результат этапа

Результатом считается код и документация, с помощью которых можно локально поднять MCP server, подключить к нему client и увидеть список доступных инструментов.
