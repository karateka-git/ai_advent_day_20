## Этап 1. Инициализация Kotlin-проекта

Статус: завершён

Цель этапа:

- развернуть минимальный Kotlin/JVM-проект;
- подключить систему сборки и зависимости;
- подготовить базовую структуру для дальнейшей реализации MCP server и MCP client.

Выполненные действия:

1. Уточнён стартовый технический набор:
   - Gradle с Kotlin DSL;
   - Kotlin/JVM;
   - официальный `modelcontextprotocol/kotlin-sdk`;
   - отдельные артефакты клиента и сервера;
   - HTTP transport как основной вектор.
2. Поднят Gradle wrapper и минимальный Kotlin/JVM-каркас проекта.
3. Добавлены базовые зависимости для Kotlin, MCP SDK, Ktor и coroutines.
4. Подготовлены отдельные точки входа для bootstrap, сервера и клиента.
5. Вынесены общие локальные настройки endpoint в `McpProjectConfig`.
6. Выполнена базовая сборка проекта через `.\gradlew.bat build`.

Принятые решения:

- основной стек проекта: Kotlin;
- сервер и клиент держать в одном JVM-проекте;
- transport- и endpoint-конфигурацию заранее не зашивать в одну большую точку входа;
- подготовить Gradle-задачи `runServer` и `runClient` до начала реализации MCP-логики.

Проверка:

- `.\gradlew.bat build` завершился успешно.

## Этап 2. Минимальный MCP Server

Статус: завершён

Цель этапа:

- поднять локальный MCP server по HTTP transport;
- зарегистрировать минимальные инструменты `ping` и `echo`;
- подготовить серверный bootstrap для следующего этапа с клиентом.

Выполненные действия:

1. Шаги этапа 2 зафиксированы в ТЗ.
2. Добавлен локальный Ktor/CIO bootstrap для MCP server с endpoint `/mcp`.
3. Добавлена фабрика `createDay16McpServer()` с `ServerCapabilities.tools`.
4. Зарегистрированы инструменты `ping` и `echo`.
5. Добавлен тест на регистрацию инструментов в server registry.
6. Выполнены `.\gradlew.bat build` и `.\gradlew.bat installDist`.
7. Отдельно проверен запуск серверного entrypoint: сервер поднимается и в логах видна регистрация инструментов.

Принятые решения:

- использовать `Application.mcpStreamableHttp(...)` из официального Kotlin SDK;
- держать bootstrap сервера отдельно от фабрики MCP server;
- использовать публичный `server.tools` как простой способ подтвердить регистрацию инструментов на этом этапе.

Проверка:

- `.\gradlew.bat build` завершается успешно;
- `createDay16McpServer().tools.keys` содержит `ping` и `echo`;
- серверный процесс стартует локально и логирует запуск на `http://127.0.0.1:3000/mcp`.

Наблюдение:

- raw проверка через `curl` пока возвращает `406 Not Acceptable`, хотя сервер стартует и создаёт session;
- этот нюанс нужно перепроверить на `Этапе 3` через официальный MCP client, чтобы не смешивать реализацию сервера и раннюю отладку клиента.

Коммиты этапа:

- `6211da0` — детализация шагов этапа 2 в ТЗ;
- `9e412d3` — добавление HTTP bootstrap для MCP server;
- `2c17e49` — регистрация инструментов `ping` и `echo`.

Следующий шаг:
