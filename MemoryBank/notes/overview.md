# Overview

Проект `ai_advent_day_16` задуман как расширяемый sandbox для знакомства с MCP.

Текущий базовый результат:

- поднять локальный MCP server;
- подключиться к нему MCP client;
- выполнить lifecycle `initialize -> tools/list`;
- вывести список доступных инструментов.

Стартовое архитектурное направление:

- по умолчанию ориентироваться на HTTP-based transport, чтобы позже можно было заменить локальный URL на удалённый endpoint без переписывания клиентского подхода;
- не смешивать transport-логику с прикладной логикой примеров;
- держать проект простым, но закладывать структуру, которую можно расширять новыми инструментами и сценариями проверки.
