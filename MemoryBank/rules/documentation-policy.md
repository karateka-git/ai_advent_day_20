# Documentation Policy

В проекте `ai_advent_day_17` для нового публичного кода и для нового важного orchestration-кода нужно по умолчанию добавлять документацию.

Правило для Codex:

- документировать новые публичные сущности и новые публичные функции;
- документировать новые нетривиальные функции, если они отвечают за MCP transport, инициализацию клиента, регистрацию инструментов, конфигурацию endpoint или другой важный orchestration;
- в README кратко фиксировать, как запустить сервер, как запустить клиент и какой transport используется.
